# Planning Archive

Dieses Dokument markiert ältere Planungsdokumente im Repository als **historische Arbeitsgrundlage**, nicht mehr als operative Führungsquelle.

## Historische Planungsartefakte

Diese Dateien waren wichtig für die Initialphase des Projekts, sind aber nicht mehr die Single Source of Truth für den laufenden Arbeitsstand.

Sie liegen jetzt unter `docs/archive/`:
- `docs/archive/backlog-and-roadmap.md`
- `docs/archive/mvp-issues.md`
- `docs/archive/roadmap.md`
- `docs/archive/project-management-and-traceability.md`
- `docs/archive/traceability-matrix.md`
- `docs/archive/github-labels.md`

## Neuer Grundsatz

Für den aktuellen Stand gilt:
- GitHub Issues und Labels sind führend
- GitHub Releases und Release Notes sind führend
- die oben genannten Markdown-Dateien dienen nur noch als historische Referenz oder Initialkonzept

## Empfehlung

Diese Dateien können später entweder:
- gelöscht,
- deutlich als archiviert markiert,
- oder in einen `docs/archive/`-Bereich verschoben werden.

Für jetzt reicht die klare semantische Umstellung: **GitHub ist die operative Wahrheit.**
